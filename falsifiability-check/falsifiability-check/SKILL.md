# 文章の反証可能性チェック（Falsifiability Check）

## 概要
提示された文章の「主張（結論）」と「根拠（理由）」を分解し、その論理がどれほど堅牢か、あるいは脆弱かを批判的に検証するためのスキルです。書き手の意図に流されず、客観的・多角的な視点から「反論の余地」を炙り出します。

## 使用方法
対象となる文章を入力し、「このスキルを使って反証可能性をチェックして」と依頼してください。以下のステップに従って分析を実行します。

## ステップ1：論理構造の抽出（事前準備）
チェックを始める前に、対象となる文章を以下の2つに分解して提示します。
- **主張・結論（C）**: 文章が最終的に言いたいこと。読み手に求めている行動や価値判断。
- **理由・根拠（R）**: 主張を支えるために提示されているデータや事実、前提。

## ステップ2：反証可能性チェックリストの適用
以下の4つのカテゴリー、16のチェック項目について検証を行い、判定（✅/⚠️/❌）を付与します。判定には以下の**汎用判定アルゴリズム**を適用し、主張の「防衛（反証可能性の確保）」がなされているかを厳格に評価します。

### 判定アルゴリズム（共通基準）
- **✅ 問題なし（堅牢）**: 主張に伴う「不可避な損失（コスト、リスク、リソース、制約）」が明示されており、かつそれを受け入れる合理的理由、または緩和策が論理的に接続されている。
- **⚠️ 懸念あり（部分的欠落）**: 負の側面には言及しているが、その評価が曖昧である。あるいは、損失を精神論や抽象的な表現で回避している（反論の隙が残っている）。
- **❌ 脆弱性あり（盲点）**: メリットや必然性のみを強調し、その裏側で確実に発生する負の側面や物理的・経済的制約に一切触れていない（現実的な実装・運用段階で崩壊するリスクが高い）。

### 1. 挑戦（Challenge）：結論を直接否定する
「Cではない」と真っ向から否定できる要素がないかを検証します。
- **結論との矛盾**: 提示された結論（C）と明確に矛盾する事実やデータはないか？
- **前提の崩壊**: 結論成立に必要な「隠れた前提条件」が崩れるケースはないか？
- **行動の不要性**: そもそもその行動をとる必要性がないと証明できないか？

### 2. 代案（Alternative）：別の選択肢を提示し相対化する
「Cではなく、C'である」と別の解を提示することで、結論の唯一性を崩せないかを検証します。
- **優れた代替案の存在**: 提示された結論以外に、より効果的・効率的な選択肢（C'）はないか？
- **トレードオフの指摘**: その結論を採用することで失われる、別の重要な価値や目的はないか？（※上記アルゴリズムに基づき、代償の隠蔽を厳しくチェックする）
- **アプローチの転換**: 問題解決の根本を変えることで、元の結論を無効化できないか？

### 3. 阻止（Undercut）：根拠となる事実そのものを否定する
「そもそも理由（R）が間違っている」と、土台を崩せないかを検証します。
- **事実誤認・データ不足**: 根拠（R）自体に誤りや証拠不十分な点はないか？
- **恣意的な「名づけ」**: 書き手に都合よく言葉が選ばれ、印象操作されていないか？
- **先決問題要求・循環論法**: 証明されていない事柄が「正しい前提」として紛れ込んでいないか？
- **根拠同士の矛盾**: 複数の根拠の背景にある思想が矛盾し合っていないか？

### 4. 反駁（Rebuttal）：理由と結論の「つながり」を断絶させる
「RだとしてもCとは限らない」と言えないかを検証します。
- **無能力の表明（実現可能性）**: 現実的な資源（時間・資金・人員）が不足しており、実行不可能ではないか？
- **自然現象化（有効性）**: 結論の行動をとっても、問題の解決や回避には繋がらないのではないか？
- **解決の代償（副作用）**: 結論を実行することで、元の問題より重大な悪影響が生じないか？
- **パースペクティヴ化（普遍性）**: 特定の状況にのみ当てはまり、全体に適用する必要はないのではないか？
- **配列による錯覚（連結の不当性）**: 独立した事実が、接続詞によって不当に結びつけられていないか？

## 出力形式
1. ステップ1の結果（CとRの抽出）
2. ステップ2のチェック結果（表形式、または箇条書き）
3. 総合評価（論理の堅牢さの度合い）
